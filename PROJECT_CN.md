# FDBCS - Fasta 数据库控制系统 (工程规格文档)

## 1. 项目概述
FDBCS 是一个轻量级、高性能的本地系统，专为管理和分析 eDNA 参考序列库而设计。它提供了一个基于 Web 的界面，用于索引大型 FASTA 文件、执行快速搜索并生成分类统计数据。

- **目标**：为生物信息学数据管理提供“本地系统 + Web 界面”解决方案。
- **目标用户**：从事 eDNA 和参考数据库工作的研究人员和生物信息学家。

---

## 2. 技术栈

### 后端 (系统核心)
- **运行时**：Node.js (ES Modules)
- **框架**：Express.js
- **数据库**：SQLite (通过 `better-sqlite3`)，用于元数据索引和快速检索。
- **文件系统**：原生 `fs` 模块，用于直接进行 FASTA 字节偏移读取 (O(1) 序列检索)。

### 前端 (Web 界面)
- **框架**：React 18 (TypeScript)
- **构建工具**：Vite
- **样式**：Tailwind CSS (Brutalist 粗犷主义设计风格)
- **可视化**：Recharts (饼图、柱状图)
- **动画**：Framer Motion
- **图标**：Lucide React

### AI 集成
- **模型**：Google Gemini (通过 `@google/genai`)
- **功能**：分类背景分析和序列功能洞察。

## 3. 系统架构

FDBCS 系统采用解耦架构，专为高性能和可扩展性而设计：

- **展示层 (Web)**：React + Tailwind CSS + Recharts，提供现代化的响应式用户界面。
- **API 层 (Node.js)**：Express 服务器，作为 Web UI 与处理/存储层之间的轻量级代理。
- **数据库操作单元 (Python)**：独立的 Unix 可执行程序 (Python 脚本)，负责所有繁重的生物信息学处理，包括 FASTA 解析、元数据匹配和统计计算。
- **存储层 (SQLite)**：持久化存储序列元数据和预计算的统计报告，确保 UI 的近乎瞬时的数据检索。

### 数据流
1. **输入**：FASTA 文件 (`db.fa`) 和元数据文件 (`Metadata.txt`) 放置在 `data/[db_name]/` 目录下。
2. **处理**：**数据库操作单元 (Python)** 解析文件，计算每个序列的字节偏移量，并生成全面的统计报告。
3. **存储**：元数据和预计算的统计信息存储在本地 SQLite 数据库中 (`storage/[db_name].index.db`)。
4. **检索**：
   - 元数据/搜索/统计：直接从 SQLite 查询 (统计信息为 O(1)，元数据为快速索引搜索)。
   - 序列内容：使用存储的字节偏移量直接从 FASTA 读取 (无需将整个文件加载到内存中)。
4. **可视化**：后端聚合分类计数；前端渲染交互式图表。

### 分类阶元系统
系统支持 9 级分类层次结构：
1. `LEVEL_0` (顶层分组)
2. `Domain` (域)
3. `Kingdom` (界)
4. `Phylum` (门)
5. `Class` (纲)
6. `Order` (目)
7. `Family` (科)
8. `Genus` (属)
9. `Species` (种)

## 4. 核心模块与功能

### 1. 数据库管理 (控制面板)
- 扫描 `data/` 目录下的数据库文件夹。
- 检测所需文件 (`db.fa`, `Metadata.txt`) 是否存在。
- 处理带有事务支持的 SQLite 索引过程，以确保性能。
- **重新初始化索引**：支持对已索引的数据库进行重置并重新构建索引，以同步最新的文件更改。

### 2. 数据库总览
- 实时统计：序列总数、唯一分类单元数、总碱基数。
- **分类阶元概览**：在物种卡片中直接显示各级阶元（域、界、门、纲等）的唯一独特单元数量。
- **层级下钻检索**：支持点击各分类层级，打开详细列表查看该层级下包含的所有唯一独特分类单元及其对应的序列条数。
- 序列长度分布：平均值、最大值、最小值。
- 前 500 条序列的预览列表。

### 3. 序列检索模块
- 跨 Accession、Header 和 Taxonomy 字段的全文本搜索。
- 单个序列的详细视图。
- AI 驱动的分类学分析。

### 4. 统计报表
- 分类阶元的视觉分布。
- 顶层 (LEVEL_0) 分布的交互式饼图。
- 涵盖所有分类层级的详细拆分模态框。

### 5. 数据库操作单元 (Bio-Ops)
- **Unix 可执行程序集成**：核心生物信息算法以独立 Python 脚本形式实现，通过系统调用执行。
- **命令行工具 (CLI)**：提供强大的统一命令行接口 (`fdbcs.py`)，支持初始化数据库、导出统计数据、删除特定序列、O(1) 提取序列、碱基组成分析及分类一致性审计等功能。详细用法请参考 [CLI_USAGE_CN.md](./CLI_USAGE_CN.md)。
- **自动化系统发育分析**：在数据库初始化过程中，自动按“目” (Order) 对序列进行分组，并调用 MAFFT 和 FastTree 进行自动比对与建树，结果保存至 `analysis_results/` 目录。
- **碱基组成分析**：基于 Python 计算全库 GC 含量、碱基频率及 AT/GC 比率。
- **分类一致性审计**：基于 Python 自动检测元数据中的分类层级错误或无效节点。
- 终端风格的结果输出界面。

## 5. 工程细节

### 目录结构
- `/data/`：用户提供的数据库文件 (输入)。
- `/storage/`：系统生成的 SQLite 索引 (内部)。
- `/src/System.tsx`：Web 界面的主要入口点。
- `/server.ts`：后端逻辑、API 路由和 Vite 中间件。
- `/DEVELOPMENT_LOG.md`：变更的按时顺序记录。
- `/PROJECT.md`：英文版工程规格文档 (单一事实来源)。
- `/PROJECT_CN.md`：本文件 (中文版工程规格文档)。

### API 规范
- `GET /api/databases`：列出可用数据库。
- `POST /api/databases/load`：触发特定文件夹的索引。
- `GET /api/search?dbName=...&query=...`：搜索元数据。
- `GET /api/overview?dbName=...`：获取摘要统计。
- `GET /api/stats?dbName=...`：获取分类阶元计数。
- `POST /api/operations/run`：执行生物信息学操作。
- `GET /api/sequence?dbName=...&accession=...`：检索序列字符串。
